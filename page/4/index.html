数据库-oracle基础

发表于 2017-05-17 | 分类于 数据库

人的一切痛苦，本质上都是对自己的无能的愤怒。

基础

  • nal(a,b)当a为Null的时候取值b，当a不为null时。
  • select * from money is null ,—–查询money字段为空值的记录
  • oracle 别名 和Mysql 一样.
  • sekect distinct money from emp —查询emp表中所有的钱的数量，去除重复。distinct这个关键字表示去重.
  • 在Oracle中单引号表示日期和字符，双引号表示列的别名。
  • between a and b 查询大于等于a小于等于b的值
  • like 模糊查询 %表示任意长度的任意字符串 _表示任意字符的一个字符串, \表示转义字符.
  • 在and语句中，where解析顺序 右—->左
  • order by 后面 +列、表达式、别名、序号 + 排序规则 asc desc
  • order by 作用于后面所有的列，先按照第一个列排序，再后面的列，desc降序 制作用于离他最近的列。
  • nulls last 表示如果有空值，空值放在最后 select * from emp order by money desc nulls last 升序时没有问题。因为在oracle数据库中，null最大
  • dual 为oracle中没有意义的空表
  • 去掉delect产生后的碎片 alter table <表名>mover 提高查询速度
  • rownum 行号 select rownum from a ， rownum永远按照默认（即使排完序后行号也不变）的顺序，rownum只能使用小于，小于等于，不能用其他。
  • top-n问题：select f from (select from a order by money desc) where rownum<=3.
  • 所有包含于select列表中，而未包含于组函数中的列，都必须包含于GroupBy子句中。包含在Group子句中的列，不一定要包含在select中。
  • 多个列的分组：先按照第一个列分组，如果相同，再按照第二个列分组，以此类推。
  • having语句：从分组的基础上，过滤。select b avg(c) from table group by b having avg(c)>2000.和where的区别是，having可以使用组函数（多行函数）。
  • group by rollup(a,b) : 报表常用。
