数据库基础-基于oracle

发表于 2017-07-27 | 分类于 数据库

人生若只如初见 何事秋风悲画扇

基础

  • 包含null的表达式都为null。
  • 空值永远不等于空值。
  • 如果集合中含有null，不能使用notiin，可以使用In
  • dual表为空表，使用在与表没关系中的select语句中。
  • ||表示连字符相当于JAVA中的+。
  • 双引号表示列的别名，单引号才表示日期和字符。
  • where查询时字符大小写敏感，日期格式敏感
  • where查询时between a and b ,含有边界，小的在钱，大的在后。
  • where查询时in(a,b)查询在ab,集合中。
  • 模糊查询Like中，%表示任意长度的任意字符，_表示一个长度的任意字符。
  • select * from user where name like ‘%_%’ escape ‘\’ : 使用escape申明一个转义字符，将_转义。
  • oracle自动开启事务，而mysql是手动开启。
  • where语句解析的顺序是从右往左。
  • order by 语句写在select语句结尾，order by + 列、序号、别名、表达式。
  • order by作用于后面所有的列，先按照第一个列拍寻，再后面的列，但是desc降序只作用于离他最近的列。
  • 日期加减上数字任为日期，数字单位是天，两个日期相减，返回的是他们之间差的天数。日期之间是不可以相加。
  • 组函数会自动滤空，使用nvl嵌套能防止这个问题。
  • 在select语句中，所有未包含在组函数中的列，应该包含在group by子句中。
  • 多个列的分组：先按照第一个列分组，如果相同，再按照第二个列，以此类推。
  • having 语句 卸载group by语句后面，过滤分组。
  • where 和having 区别 ： where语句能使用组函数，where在group by之前。为了sql性能尽量使用where语句。
  • group by rollup(a,b) = group by a,b +group by a +没有group by
  • 外连接：希望把某些不成立的记录，任然包含在最后的结果中。
  • 左外连接：当where e.a=c.a 不成立的时候，等号左边的表任然被包含在结果中。写法：where e.a(+)=c.a。又外连接相反。
  • 自链接：通过起不同的别名，将一张表看作两张表。
  • 子查询：就是select语句嵌套查询，子select语句作为父select语句的条件。
  • 子查询：子select需要加括号。
  • 一般不在子查询中排序，但是在TOP-N问题中，必须对子查询排序。
  • 单行子查询：子查询只返回一条记录，多行相反。
  • 性能方面：多表查询好于多表查询。
  • in:在集合中，any:和集合中的任意一个比较，ALL：和集合中的所有值进行比较。
  • 集合运算：参与运算的各个集合必须类型一样，列数相同
  • 采用第一个集合作为最后的表头。
  • order by 永远在最后。
  • 尽量不使用集合运算。
  • oracle中&name为占位符，和JDBC中的？一样。
  • create index [indexname] on [table] (列名字)

    函数

  • substr(a,b)从a中，取b位，substr(a,b，c)从a中，b开始取，取c位，
  • length():取字符数，lengthb()：取字节数。
  • instr(a,b)在a中，查找b。
  • round(a,b)a中保留b位小数，四舍五入。
  • select sysdate from dual :查询系统时间。
  • tochar(a,b)将a按照b的格式转化位字符串，select to-char(sysdate,’yyyy-mmydd hh24:mi:ss’)
  • months_between(a,b)返回a和b日期中相差的月数，月数是准确值。
  • add_months(a,b)在某个日期上加上b个月。
  • next_day(a,b)从a日期开始下一个星期b是多少号。
  • 转换函数to_char,to_date,to_number都接受两个参数，第一个是需要转化的字节，第二个为格式。
  • nvl(a,b)当a为null，的时候返回b，滤空函数。
  • nvl2(a,b,c)当a为空的时候返回c，否则返回b。
  • nullif(a,b)当a=b的时候，返回Null，否则返回a.
  • coalese(a,b,c….)从左到右，找到第一个不为null的值。
  • case expr when a then b when c then d 类似与if else 当a=expr时，执行b.这个是sql99标准。
  • decode(条件，’判断的值’,相对前面值执行结果，’判断的值’,相对前面值执行结果,’都不符合执行的值’) ifelse
  • oracle临时表中，会话提交，事务提交，数据会被清空。
  • 相关子查询：就是把主查询查询出来的值作为参数传到子查询里面去。
